### שלב 2 – בדיקות בדמו בלבד

- **2.1 לוח בקרה**
  - לוודא שאתה רואה:
    - כותרת החלון הנוכחי (BTC Up/Down 5m).
    - מחיר לנצח (Price to beat) + הסבר.
    - מחיר BTC נוכחי והפרש ↑/↓.
    - מחירי Up/Down בסנטים (mid + bid/ask) שנמשכים מה־orderbook.

- **2.2 אסטרטגיה בסיסית**
  - לעבור ללשונית `אסטרטגיה`.
  - לטעון פריסט **„מתחיל פשוט”**.
  - לשמור הגדרות.
  - להפעיל מצב **חצי־אוטומטי**.
  - לבדוק:
    - שמופיעות הצעות קנייה/מכירה בדמו.
    - שאישור/ביטול פקודה עובד (רואים ביומן + בדמו).

- **2.3 DCA**
  - להפעיל פריסט **„עם DCA”**.
  - להריץ כמה חלונות 5 דקות בדמו ולוודא:
    - שיש יותר מכניסה אחת (פריסות).
    - שאין פריסות חדשות בדקה האחרונה של חלון.

- **2.4 גידור**
  - להפעיל פריסט **„גידור”**.
  - לבדוק בדמו:
    - כניסה לרגל ראשונה.
    - כש־Ask Up+Down נמוך מסף → נוצרה הצעת רגל שנייה/ביצוע רגל שנייה (לפי המצב).

- **2.5 סטטיסטיקה**
  - לעצור את הבוט.
  - לעבור ללשונית `סטטיסטיקה` ולוודא:
    - שיש עקומת הון.
    - רשימת עסקאות דמו נראית הגיונית (כיוונים, מחירים).

### שלב 4 – מעבר אפשרי לכסף אמיתי (רק אם מחליטים)

- **4.1 התקנת ספריית CLOB**
  ```bash
  cd "/Users/shlomishemtov/Documents/cursor project/polymarket-bot-mac/engine"
  pip3 install -r requirements.txt
  ```
  (כולל `py-clob-client` — ראו `engine/requirements.txt`.)

- **4.1ב משתני סביבה (אופציונלי, בטרמינל לפני `uvicorn`)**
  - `POLYMARKET_PRIVATE_KEY` — חלופה להדבקת מפתח ב־UI
  - `POLYMARKET_LIVE=0` — כיבוי מיידי של שליחת פקודות לייב (המנוע ממשיך; דמו/סטטוס)
  - `POLYMARKET_SIGNATURE_TYPE` — ברירת מחדל `0` (EOA). חשבון דרך האתר לעיתים דורש `1` או `2` + `POLYMARKET_FUNDER` (כתובת הארנק) — ראו [Quickstart / Signature Types](https://docs.polymarket.com/developers/CLOB/quickstart)

- **4.2 הכנת חשבון Polymarket**
  - לפתוח / להכין ארנק או חשבון Polymarket.
  - להחליט איזה מפתח פרטי או מנגנון חתימה אתה מוכן להזין (ולהבין את הסיכון שבזה).

- **4.3 חיבור לייב באפליקציה**
  - בחלון האפליקציה:
    - לעבור למצב **LIVE**.
    - להדביק מפתח פרטי בשדה.
    - ללחוץ **„שמור מפתח לסשן”**.
  - להישאר במצב **חצי־אוטומטי** או **אוטומטי** (באוטו — עם מפתח, פקודות BUY/SELL לפי האסטרטגיה נשלחות ל־CLOB):
    - כשהמערכת מציגה פקודה בהמתנה, ללחוץ **„אשר פקודה”** — ב־LIVE הבקשה היא `POST /api/strategy/approve` עם `{ "live": true }`, הזמנה ל־CLOB ואז שיקוף לדמו לסטטיסטיקה.
  - **יתרת הדמו** במצב לייב אינה משקפת USDC אמיתי — רק מעקב לוגי אחר TP/DCA/סטטיסטיקה.

> ⚠️ **אזהרה:** מסחר בכסף אמיתי כרוך בסיכון. חובה לוודא התאמה לתנאי Polymarket ולחוק במדינה שלך לפני שמדליקים LIVE.

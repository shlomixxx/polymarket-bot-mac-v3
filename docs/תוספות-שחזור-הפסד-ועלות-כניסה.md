## 1. שחזור אחרי הפסד (Loss recovery / הגדלת השקעה)

### מטרה

מצב אופציונלי ב**אסטרטגיה הראשית** (לולאת `StrategyRunner` בלבד, לא Trigger Trader): אחרי **פירוק מופסד** בסוף חלון, להגדיל את סכום ההשקעה היעד לכניסות הבאות; אחרי **רווח** (TP או פירוק מנצח) לחזור לסכום הבסיס שמוגדר ב־`investment_usd`.

### נוסחה

- **יעד השקעה אפקטיבי**:  
  `effective_investment = investment_usd × loss_recovery_multiplier`  
  המכפיל מתחיל ב־`1.0` ונשמר בחשבון הדמו.

### איפוס

- **TP מוצלח**: `streak = 0`, `multiplier = 1.0`.
- **עסקת פירוק עם `realized_pnl > 0`**: אותו איפוס.
- **איפוס דמו / ניקוי סטטיסטיקה עם flatten**: דרך `sync_runtime_after_demo_positions_cleared` — גם streak ומכפיל מתאפסים.

### עדכון אחרי הפסד

- על כל עסקת פירוק עם `realized_pnl < 0`: `loss_recovery_streak` עולה ב־1.
- כאשר `streak > 0` ו־`streak % loss_recovery_every_n_losses == 0`, המכפיל מוכפל ב־`(1 + loss_recovery_step_pct / 100)`, עד תקרת `loss_recovery_max_multiplier`.
- אם `loss_recovery_step_pct` הוא 0 — streak עולה אבל המכפיל לא גדל (שימושי לניטור בלבד).

### היכן זה משפיע במנוע

- כניסה בלי DCA, חישוב `per` ב־DCA, וגידור (חצי מסכום — מחושב מ־יעד אפקטיבי).

### API

**POST/GET** `/api/strategy/config` — שדות קונפיג:

- `loss_recovery_enabled` (bool)
- `loss_recovery_step_pct` (float)
- `loss_recovery_every_n_losses` (int, מינימום 1)
- `loss_recovery_max_multiplier` (float, מינימום 1)

**GET** מחזיר גם (לתצוגה, מצב ריצה בדמו):

- `loss_recovery_streak`
- `loss_recovery_multiplier`

## 2. טבלת היסטוריית עסקאות — עמודת «עלות הכניסה»

### מטרה

בטבלה המקופלת/מורחבת של עסקאות לפי סשן (בטאב סטטיסטיקה), להציג בכל שורת **כניסה (BUY)** את **מחיר × חוזים** בדולרים (ברוטו לפני עמלה), כדי לראות כמה הושקע בכל שלב כניסה.

### התנהגות

- עמודה **אחרי** «מחיר», לפני «עמלה».
- רק לשורות `BUY`: מוצג `$X.XX`; בשאר סוגי העסקאות — `—`.
- Tooltip על התא: נוסחת `מחיר × חוזים`.

### קובץ

- [`src/App.tsx`](../src/App.tsx) — כותרות הטבלה, תא, והסבר קצר מתחת לכותרת «היסטוריית עסקאות».

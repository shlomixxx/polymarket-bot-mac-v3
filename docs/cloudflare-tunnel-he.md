# Cloudflare Tunnel (Quick Tunnel) — גישה מרחוק ל־UI המקומי

מסמך זה מסביר **איך מפעילים** מנהרת Cloudflare (`*.trycloudflare.com`) כדי לפתוח את ממשק הפרויקט (למשל דף השידור `?stream=7`) **מכל מקום בעולם**, בלי לפרוס לשרת קבוע.

---

## איך זה עובד (בקצרה)

1. במחשב שלך רצים **המנוע** (API על פורט **8767**) ו־**Vite** (ממשק, בדרך כלל פורט **5175** — ראה `vite.config.ts` ו־`package.json`).
2. מתקינים את **`cloudflared`** (לקוח של Cloudflare).
3. מריצים פקודה שיוצרת **מנהרה**: תעבורה מהאינטרנט מגיעה לשרתי Cloudflare, ומשם **מוצפנת** אל המחשב שלך, לכתובת המקומית של Vite.
4. Cloudflare נותן לך כתובת HTTPS זמנית, למשל:  
   `https://random-name.trycloudflare.com`  
5. הדפדפן נטען את ה־UI דרך אותה כתובת. בקשות ל־`/api` עוברות דרך **אותו שרת Vite** (פרוקסי ב־`vite.config.ts`) אל `127.0.0.1:8767`, כך שהמנוע נשאר מקומי והכל עובד כמעט כמו ב־`http://127.0.0.1:5175`.

**חשוב:** המחשב שמריץ את הפרויקט והמנהרה חייב **להישאר דלוק ומחובר לאינטרנט**. כשסוגרים את הטרמינל או את `cloudflared`, הקישור מפסיק לעבוד.

---

## התקנת `cloudflared`

### macOS (Homebrew)

```bash
brew install cloudflare/cloudflare/cloudflared
```

### בדיקה

```bash
cloudflared --version
```

---

## מה להריץ לפני המנהרה

בטרמינל נפרד (או כבר מופעל ע"י `npm run dev`):

- **מנוע** על פורט **8767** (כמו בפיתוח רגיל).
- **Vite** על הפורט המוגדר בפרויקט (למשל **5175** — בדוק ב־`package.json` וב־`vite.config.ts`).

אם Vite לא רץ, אין למה לחבר את המנהרה.

---

## הפעלת Quick Tunnel

בטרמינל (בזמן ש־Vite רץ על המחשב):

```bash
cloudflared tunnel --url http://127.0.0.1:5175
```

אם אצלך Vite על פורט אחר, החלף את `5175` בהתאם.

בפלט תופיע שורה עם כתובת HTTPS, למשל:

```text
https://something-random.trycloudflare.com
```

זו הכתובת לשיתוף. אפשר לצרף את נתיב השידור:

```text
https://something-random.trycloudflare.com/?stream=7&fit=1#stream-stats
```

---

## התאמה לפרויקט זה

- ב־`vite.config.ts` מוגדר `allowedHosts` עבור `*.trycloudflare.com` — כדי ש־Vite יקבל בקשות עם ה־Host של המנהרה.
- הפרוקסי של `/api` ו־`/ws` מפנה למנוע המקומי — כך שלא צריך מנהרה נפרדת רק ל־8767 כשגולשים דרך אותו ממשק Vite.

|------|------|
| **כתובת משתנה** | ב־Quick Tunnel השם בדרך כלל **אקראי** בכל הרצה. הקישור מהשיחה הקודמת לא מובטח שיעבוד בפעם הבאה. |

| **ייצור (production)** | לשימוש קבוע ומבוקר עדיף **פריסה** (למשל Railway) או **Named Tunnel** ב־Cloudflare עם מדיניות גישה. |

## תקלות נפוצות

- **502 / לא נטען** — ודא ש־Vite רץ על הפורט שציינת ב־`--url`.
- **חסום / CORS** — בדרך כלל נפתר עם `allowedHosts` (כבר בפרויקט). אם שינית גרסת Vite, בדוק שוב.
- **ה־API לא עונה** — ודא שהמנוע על **8767** רץ; הפרוקסי מנתב אליו רק כשגולשים דרך Vite.

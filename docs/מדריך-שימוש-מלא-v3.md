## מה הבוט עושה? {#מה-הבוט-עושה}

הבוט מסחר אוטומטי ב-**Polymarket BTC Up/Down** — שוק בינארי שבו כל חמש דקות (או 15 דקות) נפתח "חלון" חדש, ואתה מהמר: האם מחיר BTC יהיה גבוה יותר בסוף החלון מאשר בהתחלתו (Up) או נמוך (Down).

**הבוט עושה:**
- מוצא את החלון הפעיל אוטומטית
- מנתח סיגנלים (TA, CLOB, סנטימנט) ומציע כיוון
- קונה חוזים לפי הגדרות המחיר שלך
- יוצא אוטומטית ברווח (Take Profit)
- תומך ב-DCA (כניסה הדרגתית), גידור, וכניסה מחדש אחרי רווח

## מצבי הפעלה {#מצבי-הפעלה}

### OFF — כבוי (ברירת המחדל)

```
● OFF
```
- הבוט עוקב אחרי השוק אבל **לא מבצע עסקאות**
- בטוח לבדוק הגדרות ולצפות בסיגנלים
- המצב הזה נבחר בכל הפעלה מחדש (לא שמור)

---

### SEMI — חצי-אוטומטי (מומלץ למתחילים)

```
◑ SEMI
```
- הבוט מזהה הזדמנות ו**מבקש אישור** לפני כל עסקה
- ממשק מציג:
  ```
  ← אישור נדרש: קנה 25 חוזי Up @ 0.40$
  [אשר]  [דחה]
  ```
- אם לא מאשרים תוך כמה שניות — ההזדמנות עוברת
- **מתאים לתקופת לימוד** — אתה שולט אבל הבוט עושה את הניתוח

---

### AUTO — אוטומטי מלא

```
● AUTO
```
- הבוט פועל לחלוטין **ללא התערבות**
- קונה, מוכר, נכנס מחדש — הכל לפי ההגדרות
- **מומלץ רק** אחרי שבדקת בדמו שההגדרות עובדות טוב

> ⚠️ **חשוב:** כשעוברים מ-OFF ל-SEMI/AUTO, לוג האירועים מתאפס. זה מכוון — כדי להראות רק אירועים מהסשן הנוכחי.

## הגדרות האסטרטגיה {#הגדרות-אסטרטגיה}

### קטגוריה 1 — כניסה בסיסית

#### סכום השקעה ($)
**מה זה:** כמה דולרים להכניס בחלון אחד.
- ללא DCA: זה הסכום הכולל לעסקה בודדת
- עם DCA: מתחלק שווה בין כל הסלייסים
**דוגמה:** `$7.50` עם 4 סלייסים DCA = ~`$1.875` לסלייס

---

#### כניסה במחיר (סנטים) ¢
**מה זה:** המחיר המקסימלי שתשלם לחוזה.
- **1¢–99¢** (בהתאמה ל-0.01$–0.99$)
- הבוט קונה **רק אם Ask ≤ cap שלך**
- מחיר נמוך יותר = עסקה זולה יותר = יותר חוזים = רווח גדול יותר אבל הסיכוי להיממש קטן יותר

**טווחים נפוצים:**
| מחיר | משמעות |
|------|--------|
| **25¢–35¢** | שוק מוטה נגדך (סיכוי נמוך, פוטנציאל x3) |
| **40¢–50¢** | שוק מאוזן (הנפוץ) |
| **55¢–70¢** | שוק מוטה לטובתך (סיכוי גבוה, רווח קטן) |

---

#### מינימום חוזים
**מה זה:** אם החישוב (investment ÷ price) נותן פחות מהמינימום — לא נכנסים.
**ברירת מחדל:** `5` (תואם למינימום Polymarket)

---

#### חלון שוק (5m / 15m)
**מה זה:** איזה שוק BTC Up/Down לסחור:
- `5m` — חלון של 5 דקות, מהיר יותר, יותר עסקאות ביום
- `15m` — חלון של 15 דקות, יותר זמן להסתדר, פחות עסקאות

---

### קטגוריה 2 — ניהול חלון הזמן

#### דקות מינימום לכניסה
**מה זה:** כמה דקות חייבות לעבור מתחילת החלון לפני שהבוט מורשה לקנות.

```
תחילת חלון   כניסות מותרות   קפיאה   סוף חלון
|______________|_______________|_______|
0 דק'     3 דק' (min)     1 דק' מהסוף
```

**למה חשוב:** בדקות הראשונות יש תנודתיות גבוהה ומחירים לא יציבים. כניסה מוקדמת = סיכון גבוה.
**ברירת מחדל:** `3 דקות`

---

#### קפיאה בדקות אחרונות
**מה זה:** כמה דקות לפני סוף החלון הבוט **לא נכנס** לעסקאות חדשות.
**למה:** בדקה האחרונה כבר אין מספיק זמן להגיע לרווח.
**ברירת מחדל:** `1 דקה`

---

#### חסימת כניסות בינוניות
**מה זה:** אם כבר יש פוזיציה פתוחה — הבוט **לא נכנס שוב** (אלא ב-DCA).
**מומלץ:** מופעל תמיד, מונע הגדלת חשיפה בלתי מבוקרת.

---

### קטגוריה 3 — יציאה (Take Profit)

#### יעד רווח % (Take Profit)
**מה זה:** כשהרווח הלא-ממומש מגיע לאחוז הזה — הבוט מוכר.
- **10%–15%** = שמרני (יציאה מהירה, מחטיא עליות גדולות)
- **20%–35%** = מאוזן
- **50%+** = אגרסיבי (מחכה לתנועות גדולות)

**נוסחת חישוב:**
```
רווח % = (bid_נוכחי - מחיר_קנייה_ממוצע) ÷ מחיר_קנייה_ממוצע × 100
```

---

### קטגוריה 4 — DCA (Dollar-Cost Averaging)

**DCA = כניסה הדרגתית.** במקום לקנות הכל בבת אחת, קונים בחתיכות קטנות לאורך זמן.

#### מספר סלייסים
**מה זה:** לכמה חלקים לחלק את הכניסה.
**דוגמה:** `4` סלייסים × `$1.875` = `$7.50` כולל
**שימוש:** לשוק תנודתי — קניות נפרדות ממצעות את מחיר הכניסה.

---

#### מרווח בין סלייסים (שניות)
**מה זה:** כמה שניות להמתין בין סלייס לסלייס.
**דוגמה:** `30` שניות × `4` סלייסים = 2 דקות לסיום הכניסה
> ⚠️ ודא ש-`(סלייסים-1) × מרווח < משך_חלון - דקות_מינימום - קפיאה`

---

#### הנחת DCA %
**מה זה:** כל סלייס מנסה לקנות **X% מתחת** למחיר המילוי הקודם.
**דוגמה:** מילוי ראשון ב-0.35$, הנחה 5% → cap סלייס שני = 0.35 × 0.95 = 0.3325$
**מומלץ:** `2%–5%` לשוק רגיל, `0` כשלא רוצים הגבלה נוספת.

---

#### DCA Override % (יציאה מוקדמת)
**מה זה:** מאפשר TP מוקדם גם לפני השלמת כל הסלייסים, אם הרווח חצה סף גבוה.
**למה:** אם BTC זז חזק לטובתך כבר בסלייס 2, עדיף לצאת עכשיו מאשר להמתין.
**ברירת מחדל:** `50%`

---

### קטגוריה 5 — גידור (Hedge)

**גידור = קניית שני הצדדים בו-זמנית.**

#### הפעלת גידור
כשמופעל: הבוט קונה גם Up וגם Down בחלון אחד.
**כניסה מותרת רק** כש: `ask_up + ask_down ≤ hedge_combined_ask_max`

#### מחיר משולב מקסימלי
**ברירת מחדל:** `0.98` — כלומר, אם ask_up + ask_down = 0.95$, אפשר לגדר.
**מינימום רווח תיאורטי:** `1.00 - 0.98 = 0.02$` לחוזה (לפני עמלות).

> ⚠️ גידור **לא מבטיח רווח** — עמלות שני הצדדים יכולות לאכול את המרווח.

### קטגוריה 6 — כניסה מחדש

**מומלץ:** מופעל כשהחלון עדיין ארוך (15m).

**ברירת מחדל:** `8 שניות`

**דוגמה:** TP=20%, קרוב TP=2% → alert כשרווח ≥ 18%

## לשונית סיגנלים 📡 {#לשונית-סיגנלים}

**זהו הכלי המרכזי לכיוון ההשקעה.** הלשונית מאחדת 4 מקורות מידע לציון ביטחון אחד.

### כרטיס ההמלצה הראשי

```
          המלצת כיוון
         ⬆ UP
     ביטחון: 73.2%

  Up ████████░░ 73%   Down ░░░░░░░░ 27%
```

- **⬆ UP / ⬇ DOWN** — כיוון מומלץ
- **≡ ניטרלי** — אין כיוון ברור, **לא כדאי להיכנס**
- **ביטחון > 60%** = המלצה פעילה; מתחת לזה = ניטרלי

### 4 מקורות הסיגנל

#### 📊 ניתוח טכני (TA) — משקל 40%

| מדד | משמעות | אות חיובי (Up) |
|-----|--------|----------------|
| **RSI(14)** | מומנטום 0-100 | RSI > 55 |
| **EMA 9/21** | מגמה קצרה vs. ארוכה | EMA9 > EMA21 |
| **מומנטום 3m** | שינוי % ב-3 דקות | > +0.05% |
| **מומנטום 5m** | שינוי % ב-5 דקות | > +0.08% |

**RSI קריאה מהירה:**
```
RSI > 70  = קנייה יתר (Down עלול)
55-70     = מומנטום עולה → Up
45-55     = ניטרלי
30-45     = מומנטום יורד → Down
RSI < 30  = מכירה יתר (Up עלול)
```

---

#### 📖 CLOB Imbalance — משקל 30%

**מה זה:** בספר הפקודות (orderbook) של Polymarket — האם יש יותר קונים ל-Up או ל-Down?

```
Up token:   Bid depth $1,240  vs.  Ask depth $480   → imbalance +0.44
Down token: Bid depth $380    vs.  Ask depth $920   → imbalance -0.42
Net score = 0.44 - (-0.42) = 0.86 → סיגנל Up חזק
```

**פרשנות:**
- **imbalance > 0** = יותר קונים (bid > ask) = "smart money" קונה צד זה
- **net_score > 0** = Up מועדף

---

#### 🕓 היסטוריה — משקל 15%

**מה זה:** מסד נתונים SQLite מקומי שנבנה **בזמן ריצה**.
שומר: מי ניצח בכל חלון, מחיר BTC, שעה UTC, יום בשבוע.

```
היסטוריה 14:00 UTC — Up ניצח 62% (21 חלונות)
```

> **הערה:** בהפעלה ראשונה אין נתונים היסטוריים. הנתונים יצטברו לאחר שהבוט יעקוב אחרי חלונות שנפתרו. מומלץ להפעיל את הבוט כמה שעות ב-demo לפני שסומכים על הסיגנל ההיסטורי.

**איך מתעדים תוצאות:** שלח בקשה ל-API לאחר כל חלון:
```
POST /api/history/record
{ "epoch": 1700000000, "slug": "btc-updown-5m-...", "side_won": "Up",
  "btc_open": 42000, "btc_close": 42150, "window_sec": 300 }
```
*(בגרסה עתידית — יתוכנת אוטומטית)*

**מקור 1: Binance Funding Rate (BTCUSDT Perpetual)**

| שיעור | משמעות | סיגנל |
|-------|--------|-------|
| > +0.05% | over-leveraged longs, ירידה סביר | Down |
| < -0.05% | over-leveraged shorts, עלייה סביר | Up |
| בין -0.05% ל-+0.05% | מאוזן | ניטרלי |

**מקור 2: Fear & Greed Index (alternative.me)**

| ערך | סיווג | סיגנל |
|-----|-------|-------|
| 0–25 | Extreme Fear | Up (bounce) |
| 26–50 | Fear | ניטרלי |
| 51–75 | Greed | ניטרלי |
| 76–100 | Extreme Greed | Down (reversal) |

## מונחון {#מונחון}

| מונח | הסבר |
|------|------|
| **Epoch** | זמן Unix (שניות) — זיהוי ייחודי לכל חלון |
| **Token Up / Token Down** | מזהי החוזים בשוק הנוכחי |
| **CLOB** | Central Limit Order Book — ספר הפקודות של Polymarket |
| **Bid** | המחיר הגבוה ביותר שמישהו מוכן לשלם (קונה) |
| **Ask** | המחיר הנמוך ביותר שמישהו מוכן למכור (מוכר) |
| **Spread** | הפרש Ask - Bid |
| **DCA** | Dollar-Cost Averaging — כניסה הדרגתית |
| **TP** | Take Profit — יציאה בהגעה לרווח יעד |
| **Imbalance** | הטיה בין bid depth לask depth בספר הפקודות |
| **RSI** | Relative Strength Index — מדד מומנטום (0-100) |
| **EMA** | Exponential Moving Average — ממוצע נע מעוצב |
| **ATR** | Average True Range — מדד תנודתיות |
| **Funding Rate** | שיעור מימון בחוזים עתידיים — מדד סנטימנט |
| **Fear & Greed** | מדד פחד-חמדנות שוק הקריפטו (0-100) |
| **Mark-to-Market** | הערכת שווי פוזיציה לפי מחירי שוק עכשיו |
| **Stale** | נתונים ישנים — CLOB לא עודכן לאחרונה |
| **Session** | קבוצת עסקאות בחלון אחד (מאוגדות ב-session_id) |

## עמודת "סיבה"

עמודת **"סיבה"** מתעדת **מדוע** הבוט ביצע את הפעולה — אילו תנאים התקיימו.

---

### בכניסות (Entry)

**מבנה כללי:**
```
entry_ok:צד gate=שער ask_vs_target=X%
```

| רכיב | מה הוא אומר |
|------|-------------|
| **entry_ok** | תנאי הכניסה הכללי התקיים |
| **צד** | `Up` או `Down` — הצד שנבחר |
| **gate** | שער הזמן — `ok` / `intermediate` / `freeze` |
| **ask_vs_target** | ההפרש בין מחיר ה־Ask הנוכחי ליעד הכניסה באחוזים |

#### ask_vs_target — איך לקרוא אותו

| ערך | משמעות | דוגמה |
|-----|---------|-------|
| **מספר שלילי** | Ask **נמוך** מיעד הכניסה — קונים במחיר טוב | `-4%` = Ask נמוך ב־4% מיעד |
| **0%** | Ask שווה ליעד | קנייה בדיוק במחיר היעד |
| **מספר חיובי** | Ask **גבוה** מיעד — קונים יקר יותר | `+3%` = Ask גבוה ב־3% (לא מומלץ) |

**במלים פשוטות:**  
מינוס = **הנחה**. פלוס = **יקר יותר**.

---

### דוגמאות לכניסות

#### כניסה ראשונה
```
entry_ok:Up gate=ok ask_vs_target=-4.00%
```
- נכנסו ל־**Up**
- שער הזמן פתוח (**ok**)
- Ask נמוך ב־4% מיעד הכניסה — קיבלנו הנחה של 4%

#### DCA — פריסה שנייה
```
entry_ok:Up gate=ok ask_vs_target=-28.00%
```
- המחיר ירד — הבוט קנה שוב
- Ask עכשיו נמוך ב־**28%** מיעד הכניסה
- DCA מונע תלות במחיר אחד — מפזר סיכון

#### DCA — פריסה אחרונה לפני אזור ביניים
```
entry_ok:Down gate=ok ask_vs_target=-40.00%
```
- פריסה נוספת — מחיר Down נמוך מאוד
- 40% מתחת ליעד = מחיר טוב מאוד לקנייה

---

### ביציאות (TP — Take Profit)

**מבנה:**
```
TP צד: upnl=X% >= יעד%
```

| רכיב | מה הוא אומר |
|------|-------------|
| **TP** | יציאה ברווח (Take Profit) |
| **צד** | Up או Down |
| **upnl** | רווח לא ממומש (unrealized PnL) ברגע היציאה |
| **>= יעד** | יעד הרווח שהגדרת (למשל 20%) |

---

### דוגמאות ליציאות TP

```
TP Up: upnl=51.44% >= 20.00%
```
- יצאנו מ־**Up** ברווח
- הרווח הלא ממומש היה **51.44%** — מעל יעד 20%
- הבוט מכר אוטומטית כשהגענו ליעד

```
TP Down: upnl=66.81% >= 20.00%
```
- יצאנו מ־**Down** ברווח 66.81%
- עברת את יעד 20% — TP הופעל

---

### יציאות EXPIRE

כשהחלון נגמר **בלי** TP:

- הבוט לא מכר — הפוזיציה נסגרת אוטומטית (EXPIRE)
- התוצאה נקבעת לפי השוק — לרוב **הפסד מלא**
- הסיבה תופיע כ־`EXPIRE_0 rollover` כשהחלון עבר

## עמודת Gate

**Gate** = שער הזמן — מתי מותר להיכנס.

| ערך | משמעות | כניסות חדשות? | מכירות (TP)? |
|-----|---------|----------------|---------------|
| **ok** | יש מספיק זמן בחלון | ✓ כן | ✓ כן |
| **intermediate** | אזור ביניים (1–3 דק' לסיום) | ✗ לא | ✓ כן |
| **freeze** | דקה אחרונה | ✗ לא | ✓ כן |

בצד יופיע לעיתים גם זמן נשאר, למשל: `(4.99 דק')` או `(1.68 דק')`.

---

### דוגמאות ל־Gate

| Gate | משמעות |
|------|---------|
| `ok (4.99 דק')` | נשארו כמעט 5 דקות — כניסות מותרות |
| `ok (3.12 דק')` | נשארו כ־3 דקות — עדיין בתחום הכניסה המותר |
| `intermediate (2.15 דק')` | אזור ביניים — אין כניסות חדשות, רק ניהול פוזיציה קיימת |
| `intermediate (1.68 דק')` | נשארו פחות מ־2 דקות — ביניים |
| `freeze (0.45 דק')` | דקה אחרונה — אין כניסות |

## שאר העמודות

| עמודה | מה היא מציגה |
|-------|---------------|
| **חלון 5 דק'** | השעה שבה החל החלון (למשל 04:20) |
| **זמן** | השעה המדויקת של הפעולה |
| **פעולה** | כניסה / יציאה (TP) / EXPIRE |
| **צד** | Up או Down |
| **חוזים** | מספר החוזים שנקנו או נמכרו |
| **מחיר** | מחיר העסקה (0–1, בייצוג עשרוני) |
| **עמלה** | העמלה המשוערת |
| **רווח** | רווח/הפסד ממומש (בדרך כלל רק ביציאה) |
| **שיא** | רווח לא ממומש מקסימלי בזמן ההחזקה |
| **שפל** | רווח לא ממומש מינימלי (יכול להיות שלילי) |

## שיא ושפל

**שיא** = הרווח הגבוה ביותר שהופיע על הפוזיציה לפני היציאה.  
**שפל** = הרווח הנמוך ביותר (יכול להיות מינוס — הפסד זמני).

### איך זה מחושב

השיא והשפל מחושבים **על כל הפוזיציה** — לא פר־קנייה בודדת:

| מונח | הסבר |
|------|------|
| **ממוצע כניסה** | ממוצע משוקלל של כל הכניסות (כולל DCA) — כל קנייה מורידה/מעלה את הממוצע לפי כמות החוזים |
| **רווח לא ממומש** | `(ערך נוכחי − עלות ממוצעת) / עלות ממוצעת × 100%` |
| **שיא** | ערך מקסימלי של הרווח הלא ממומש לאורך זמן ההחזקה |
| **שפל** | ערך מינימלי של הרווח הלא ממומש |

**למה מופיעים לפעמים מעל 100%?**  
האחוזים האלה הם **תשואה ביחס לעלות** (ROI / רווח לא ממומש), **לא** מחיר החוזה. מחיר ליחידה בפולימרקט נשאר בדרך כלל בטווח **0.01–0.99$**. אם קנית בזול (למשל 0.10$) וה-bid עלה ל-0.30$, הרווח היחסי לעלות הוא +200% — למרות שהמחיר עדיין מתחת לדולר.

**במלים פשוטות:** אם קנית 3 פעמים (DCA) — הבוט לא מחשב שיא/שפל לכל קנייה בנפרד, אלא לפוזיציה המשולבת לפי הממוצע המשוקלל של כל הכניסות.

**דוגמה:**  
שיא 53.1% | שפל -18.8%

- בזמן ההחזקה הרווח טיפס עד **53.1%**
- ירד עד **-18.8%** (הפסד זמני)
- ביציאה — TP ב־51.44%

זה עוזר להבין את תנודתיות העסקה — האם הייתה קרובה להפסד לפני שהתהפכה.

### שיא ושפל פוטנציאליים (אחרי היציאה)

בממשק מוצגים **שני ניסוחים** לאותו מסלול bid אחרי TP:

1. **מול מחיר יציאה** — הפרש ב־¢ וב־% **ביחס למחיר שבו מכרת** (המדד האינטואיטיבי ל"מה עוד היה אפשר" או "מה נמנעת").
2. **מול עלות כניסה** — אחוזים **ביחס לאותה עלות ממוצעת** כמו בזמן החזקה (אותו בסיס כמו לפני המכירה).

רק **ביציאות TP** — כשמכרת לפני סיום החלון:

| מונח | הסבר |
|------|------|
| **שיא פוטנציאלי** | שוב — **תשואה % מול אותה עלות**, לא מחיר החוזה: המקסימום שהיית מגיע אליו אם נשארת **אחרי** ה-TP עד סיום החלון |
| **שפל פוטנציאלי** | המינימום של אותה תשואה % היפותטית אחרי היציאה |

**במלים פשוטות:** מכרת ב־TP, אבל ה-bid המשיך לנוע. שיא/שפל פוטנציאליים מראים איך **הייתה נראית התשואה %** (מול עלות) אילו נשארת בפוזיציה — לא "כמה עולה מחיר המניה" בדולרים.

**ב־EXPIRE** אין שיא/שפל פוטנציאליים — העסקה נסגרה עם סיום החלון.

## דוגמאות מלאות

### דוגמה 1: DCA עם TP מוצלח

| פעולה | מחיר | חוזים | Gate | סיבה |
|-------|------|-------|------|------|
| כניסה | 0.48 | 124 | ok (4.99 דק') | entry_ok:Up gate=ok ask_vs_target=-4.00% |
| כניסה | 0.46 | 135 | ok (4.85 דק') | entry_ok:Up gate=ok ask_vs_target=-6.00% |
| כניסה | 0.40 | 156 | ok (4.70 דק') | entry_ok:Up gate=ok ask_vs_target=-28.00% |
| כניסה | 0.38 | 164 | ok (4.55 דק') | entry_ok:Up gate=ok ask_vs_target=-34.00% |
| כניסה | 0.32 | 194 | ok (4.40 דק') | entry_ok:Up gate=ok ask_vs_target=-40.00% |
| יציאה (TP) | 0.57 | 1072 | intermediate (1.68 דק') | TP Up: upnl=51.44% >= 20.00% |

**מה קרה:**
1. כניסה ראשונה ב־0.48 — Ask היה 4% מתחת ליעד
2. DCA — המחיר ירד, הבוט קנה שוב ב־0.46, 0.40, 0.38, 0.32
3. ממוצע הכניסה ירד — יותר חוזים באותו כסף
4. המחיר עלה ל־0.57 — רווח 51.44%, מעל יעד 20%
5. הבוט מכר — TP. ה־Gate כבר intermediate כי נשארו פחות מ־3 דקות, אבל TP תמיד מותר

## טבלת סיכום — סיבות נפוצות

| סיבה | מתי מופיעה |
|------|------------|
| `entry_ok:Up gate=ok ask_vs_target=-X%` | כניסה ל־Up כשהמחיר מתחת ליעד |
| `entry_ok:Down gate=ok ask_vs_target=-X%` | כניסה ל־Down כשהמחיר מתחת ליעד |
| `TP Up: upnl=X% >= 20.00%` | יציאה ברווח מ־Up — הרווח עבר את היעד |
| `TP Down: upnl=X% >= 20.00%` | יציאה ברווח מ־Down |
| `EXPIRE_0 rollover` | מעבר חלון — פוזיציות מהחלון הקודם נסגרות |

---

## קישור להגדרות

| סיבה | הגדרה קשורה |
|------|-------------|
| ask_vs_target | כניסה במחיר (¢), DCA בהנחה % |
| gate | דק' מינימום לכניסה, קפיאה בדקה אחרונה, אזור ביניים |
| upnl >= X% | יעד רווח % (נטו משוער) |
| TP / EXPIRE | DCA override — מאפשר TP גם בלי DCA מלא כשהרווח מעל X% |

## איך להפעיל את הבוט (מומלץ — עם לוגים מלאים)

### דרך א׳ — דאבל־קליק ב־Finder (הכי פשוט)

1. פתח את תיקיית הפרויקט `polymarket-bot-mac`.
2. לחץ פעמיים על **`run-bot.command`** (או **`run-bot-with-logs.command`** — אותה פעולה).
3. אם macOS חוסם: **System Settings → Privacy & Security** ואשר הרצה, או לחץ ימין → **Open** בפעם הראשונה.

נפתח טרמינל, יוצרת תיקיית לוגים, ומריצה את הבוט.

### דרך ב׳ — מהטרמינל

```bash
cd "/Users/shlomishemtov/Documents/cursor project/polymarket-bot-mac"
./scripts/run-with-logs.sh
```

מה קורה כאן:

- נוצרת תיקייה: **`logs/runs/YYYY-MM-DD/HH-MM-SS/`**
- מוגדר **`LOG_RUN_DIR`** — המנוע כותב לשם קבצי JSON/יומן/עסקאות
- רצים במקביל: **מנוע (פורט 8765)** + **Vite (5173)** + **חלון Electron**
- כל הפלט מהטרמינל נשמר ב־**`combined.log`** (עם `tee`)

### לסיום

לחץ **Ctrl+C** בטרמינל. נכתב קובץ **`session_end.json`** בתיקיית הריצה.

## איפה הלוגים ומה חשוב

| קובץ / תיקייה | תיאור |
|----------------|--------|
| **`logs/runs/<תאריך>/<שעה>/`** | תיקיית הריצה |
| **`combined.log`** | כל הפלט מהטרמינל (מנוע + Vite + Electron) |
| **`meta.json`** | שם ריצה, זמן, סביבה |
| **`strategy_journal.txt`** | יומן אסטרטגיה (מומץ לניתוח) |
| **`trades.json`**, **`trades_summary.txt`** | עסקאות וסיכומים |
| **`run_diagnostics.txt`** | אבחון (ticks, פוזיציות) |
| **`session_end.json`** | נכתב בסיום (אחרי Ctrl+C) |

פירוט נוסף: **`logs/README.md`** בתוך הפרויקט.

### בדיקת נתיב הלוגים מהמנוע

כשהמנוע רץ: בדפדפן פתח  
`http://127.0.0.1:8765/api/logs/run-dir`

## תקלות נפוצות

| בעיה | מה לעשות |
|------|-----------|
| `command not found: npm` | התקן Node.js מ־https://nodejs.org — ואז `npm install` שוב |
| `ModuleNotFoundError` ב־Python | `cd engine && pip3 install -r requirements.txt` |
| חלון Electron לא נפתח | חכה שהמנוע ו־Vite יעלו; או בדוק ב־`combined.log` שגיאות |
| אין חיבור ל־API | ודא שהמנוע רץ על `127.0.0.1:8765` |
